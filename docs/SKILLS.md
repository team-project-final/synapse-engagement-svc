# Project Skills

## History 작성 스킬

작업을 마친 뒤에는 `docs/project-management/history/` 아래에 오늘 날짜의 history 파일을 새로 만든다.

- 파일명: `YYYY-MM-DD-간략한-작업명.md`
- 제목: 날짜가 아니라 `# 짧은 작업 주제` 형식으로 적는다.
- 본문: 작업일, 작업 내용, 검증, 남은 이슈를 구분해 자세히 적는다.
- 작업 내용에는 실제로 변경한 코드/문서/테스트와 왜 바꿨는지까지 bullet로 남긴다.
- 여러 작업을 한 날에도 사용자가 별도 요청하면 새 파일을 만들고, 파일명이 겹치지 않게 작업명을 다르게 쓴다.
